# MADI Mobility AI — Optimized Edition v4

Оптимизированная версия коммерческого проекта по теме:
**«Проектирование интеллектуальной платформы для управления городской мобильностью (MaaS) на основе мультиагентного подхода с прогнозированием спроса и оптимизацией логистических потоков»**.

Текущая редакция сфокусирована на **автомобильно-пешеходной мобильности внутри Одинцово** и упрощена для MVP/демонстрации без агентной подсистемы и железнодорожного контура.

## Что изменено в v4

### Сохранено из оптимизации
- нет `agents.js`
- нет расписаний электричек и ЖД-логики
- нет лишних режимов транспорта
- код разделен по слоям: UI / routing / traffic heuristics / backend meta

### Добавлено по новому ТЗ
- более выразительная анимация добавления точки **A** и **B**
- рабочий полигон Одинцово с ограничением постановки точек
- затемнение карты вне рабочей области
- возврат отображения пробок на карте
- ETA и время прибытия с учетом дорожной загрузки
- режим **мультимаршрута** с несколькими автомобильными альтернативами
- выбор оптимального маршрута из альтернатив
- подготовка архитектуры под будущую монетизацию PRO-функций
- подготовка backend/frontend структуры под будущую интеграцию с **n8n + LLM**

## Архитектурная идея

### Базовый бесплатный слой
- постановка точек A/B
- авто и пеший режим
- оценка времени, расстояния и последней мили
- визуализация пробок

### Будущий PRO-слой
- мультимаршрут
- интеллектуальный выбор лучшей альтернативы
- рекомендации по коммерческой оптимизации поездки
- диалоговый маршрутный ассистент через n8n / LLM

## Примечание по границе Одинцово

Сейчас используется **рабочий контур проекта**, чтобы сразу ограничить сценарии демонстрации. При необходимости его можно заменить на официальный GIS-полигон, не меняя архитектуру фронтенда.


## Техническая база маршрутизации

- построение маршрутов переведено на **Yandex MultiRoute**
- автомобильные маршруты учитывают `avoidTrafficJams` и могут возвращать до 3 альтернатив
- пешеходный маршрут строится тем же API в режиме `pedestrian`
- ограничение рабочей зоны реализовано через `geometry.Polygon.contains()` и `restrictMapArea`
